Projects

NoobWerkz Engine

This thing is my pet. I wrote this from the ground up in C++, and have taken out at least three times as much code as the project contains. This has been a very good project for someone wanting to tame complexity into elegance. It remains my go-to choice for experimental code needing low-level hardware access. It is very, very bare-bones, but as features it has: Full OpenGLES 3 support, a an extremely flexible "gets out of your way" component/entity system, excellent all-purpose shaders that only take up constant space of texture memory (just tweak the parameters to obtain totally different results), no RTTI/exceptions, and no deep inheritance structures: The vast majority of inheritance is as pure virtual functions that get final'd at definition-time. It also natively supports fun peripherals such as the 3DMouse and the Wacom tablet. Due to its data-oriented loop that preallocates the most possible it is already very stable, but I'd like to rewrite all parts of it that live on a gameserver to Ada/SPARK. I am migrating towards being a framework.

AutoGentoo

I use Gentoo. A lot. AutoGentoo is a set of Python scripts that do the repetitive work of building, packing, and compressing immutable system images. The generated kernels have rootfs packed right in as a linked initramfs. Will soon be using a sane and portable alternative to ZFS, plus functionality for packing filesystems for subsequent overlaying.

At this point it is mostly feature-complete and does what it should do. There will be a few backend changes and minor feature upgrades.

Fogget About It

This was my final year university project. It is an automated agricultural management system written in Java. It specifies a data format, an MQTT-based messaging system, a server-based frontend that runs on a Pi-like system (or bigger/better) and allows you to view state and update settings. It also has testing harnesses and scored me 85% for the class, so I'm pretty happy with it. It also does not leak any references, nor memory.
