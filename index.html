Projects

NoobWerkz Engine

This thing is my pet. I wrote this from the ground up in C++, and have taken out at least three times as much code as the project contains. This has been a very good project for someone wanting to tame complexity into elegance. This thing hasn't seen updates in while, but remains my goto-choice for code that I know will work well. It is very, very bare-bones, but as features it has: Full OpenGLES 3 support, a an extremelt flexible "gets out of your way" component/entity system, excellent all-purpose shaders that only take up constant space of texture memory (just tweak the parameters to obtain totally different results), no RTTI/exceptions, and no deep inheritance structures: The vast majority of inheritance is as pure virtual functions that get final'd at definition-time. It also natively supports fun peripherals such as the 3DMouse and the Wacom tablet.

It is currently still in its early teenage years: It needs refinement and additional abilities. Due to its data-oriented loop that preallocates the most possible it is already very stable, but I'd like to rewrite all parts of it that live on a server in a memory-safe language such as Rust or Ada/SPARK.

AutoGentoo

AutoGentoo is a set of Python scripts that do the repetitive work of building, packing, and compressing immutable system images. The generated kernels have rootfs packed right in as a linked initramfs. Will soon be using a sane and portable alternative to ZFS, plus functionality for packing filesystems for subsequent overlaying.

At this point it is mostly feature-complete and does what it should do. There will be a few backend changes and minor feature upgrades. Linux-wise, I believe Exherbo may fit my needs a bit more, as it seems to solve a few of the problems this toolset fixes. This fun little project may even become obsolete for me sooner than expected! Even if it does, I hope to leave it well-enough to make others happy to use it.

Fogget About It

This was my final year university project. It is a agricultural tool written in Java, specialized towards collecting and updating information from embedded systems running fogponics equipment. More specifically it specifies a data format, an MQTT-based messagfing system, a server-based frontend that runs on a Pi-like system (or bigger/better) and allows you to view state and update settings. It also has testing harnesses and scored 85% for the class, so I'm pretty happy with it. It also does not leak any references, nor memory.
